# Dao Vision

## 1. What is Dao?

**Dao is an AI-native personal workspace for developer growth.**

Dao helps developers manage their learning, projects, tasks, notes, workflows, and long-term technical growth. It starts from the student stage, supports internship and job-hunting preparation, and gradually evolves into a personal engineering operating system for professional developers.

Dao is not designed as a short-term productivity tool. It is designed as a long-term growth system.

## 2. Name Meaning

In Chinese, “Dao” means far more than “road” or “path”.

It can also mean:

- principle
- method
- order
- discipline
- long-term practice
- the underlying way of understanding and doing things

Programming growth follows the same pattern.

Becoming a better developer is not a straight line. It is a long process of learning, building, failing, reflecting, and gradually forming your own engineering methodology.

Dao exists to support that process.

## 3. Tagline

```txt
Dao — Your path to becoming a better developer.
```

## 4. Target Users

### Primary users

```txt
Student developers who are learning software engineering, building projects, preparing for internships, and preparing for job hunting.
```

### Future users

```txt
Junior engineers, independent developers, and developers who maintain long-term side projects or personal knowledge systems.
```

## 5. Core Product Positioning

Dao is positioned as:

```txt
An AI-native, local-first developer growth workspace.
```

Its product direction combines:

- project management
- task management
- technical notes
- learning records
- developer workflow
- AI context
- personal knowledge accumulation
- optional stage-based extensions

## 6. Core Product Belief

```txt
A developer’s growth should be continuously accumulated, structured, and reflected.
```

Most productivity tools focus on immediate task completion.

Dao focuses on long-term growth.

Every note, project, task, interview record, learning plan, and development log can become part of the user’s personal engineering context.

Over time, Dao should become more useful because it understands the user’s growth path, habits, projects, and knowledge base.

## 7. Product Personality

Dao should feel:

- calm
- focused
- crafted
- reliable
- developer-native
- modern
- long-term
- quietly intelligent

Dao should not feel:

- like a generic todo app
- like a Notion clone
- like an AI wrapper
- like a short-term job-hunting tool
- like a flashy productivity gimmick

## 8. Product Principles

### 8.1 Local-first

The user’s core data should be usable locally.

The first version should work without cloud sync.

Cloud sync is an enhancement, not the foundation.

### 8.2 Growth-first

The product should serve long-term developer growth rather than short-term task completion only.

### 8.3 Developer-native

Dao should feel natural to developers.

Important interactions should include:

- keyboard shortcuts
- command palette
- markdown
- project-based organization
- search
- future GitHub integration
- AI context based on real user data

### 8.4 Core + Extensions

Dao should keep its core simple.

Stage-specific features should be provided through official extensions.

Examples:

- interview extension
- LeetCode extension
- GitHub extension
- daily review extension
- AI extension

### 8.5 AI should be contextual

AI in Dao should not be a generic chat box.

AI should work with the user’s own context:

- projects
- notes
- tasks
- activity logs
- learning records
- interview preparation
- code-related knowledge

## 10. MVP Definition

The MVP should answer one question:

```txt
Can Dao become a developer’s daily workspace for managing projects, tasks, notes, and growth records?
```

The first usable loop:

```txt
Open Dao
Create a workspace
Create a project
Create tasks
Write notes
Search across content
Use command palette for quick actions
Review recent activity
```

## 11. MVP Scope

### Included

- workspace
- project
- task
- note
- dashboard
- search
- command palette
- activity log
- settings

### Deferred

- AI Agent
- cloud sync
- mobile app
- GitHub integration
- LeetCode integration
- third-party plugin marketplace
- collaboration
- team workspace

## 12. Success Criteria

Dao MVP is successful when:

- the desktop app can be opened and used daily
- projects, tasks, and notes are persisted locally
- search is useful enough for real usage
- command palette improves navigation speed
- the product feels like a developer tool
- the architecture can support future AI and extension features

## 13. Long-term Vision

Dao should become a place where years of developer growth accumulate naturally.

It should help users answer questions like:

- What have I been building?
- What have I learned?
- Which projects show my strongest skills?
- What should I improve next?
- What knowledge have I already collected?
- How has my engineering ability evolved over time?

Dao is not only a tool for writing code.

It is a system for becoming a better developer.
